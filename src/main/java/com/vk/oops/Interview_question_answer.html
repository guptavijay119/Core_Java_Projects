Checked Exceptions ✅ Definition: Exceptions that are checked at compile-time. 👉 The compiler forces you to handle them using: try-catch OR throws ✅ Examples: IOException SQLException FileNotFoundException 2. Unchecked Exceptions ✅ Definition: Exceptions that occur at runtime and are not checked at compile-time. 👉 Handling is optional ✅ Examples: NullPointerException ArrayIndexOutOfBoundsException ArithmeticException 🎯 One-Line Interview Answer 👉 Polymorphism allows us to perform a single action in different ways, achieved via method overloading (compile-time) and method overriding (runtime). ✅ Definition (Best for Interview) 1. Compile-Time Polymorphism (Method Overloading) polymorphism means “one object, many forms.” In Java, it allows a method or object to behave differently based on the context. 👉 In simple terms: Same method name → different behavior Runtime Polymorphism (Method Overriding) ⭐ (Most Important) Achieved using Inheritance + Method Overriding
